2. BUG 1 FIX — Jason Ignoring Player Speech
   File: server/server.ts
   Root cause: sendText() (uses sendClientContent, bypasses VAD) fired for
   every chained_auto step during rapid chains (e.g., park sequence 12→15).
   8+ forced text turns drowned out player audio via sendRealtimeInput.

   Fix: sendText(autoplayText) and injectSceneContextIntoJason() now only
   fire on hold_for_input steps or keyword triggers. Chained_auto steps
   skip both, leaving Jason free to listen to the player.

FRONTEND CHANGES (myceliainteractive)
================================================================================

4. BUG 2 FIX — Glitch Effect Persisting Forever
   File: app/ls/game/useGameHudGeneralEffects.ts
   Commit: 5757231
   Deployed: Cloudflare Pages version 91abe13c-8da7-4914-be70-36c67a8b5606

   Root cause: React useEffect cleanup race condition. The hud_glitch handler
   set a 1.5s timer to call setGlitchClass(null), but the useEffect cleanup
   cleared that timer when lastEvent changed (hud_glitch → scene_change).
   The CSS animation with "infinite" duration then persisted forever.

   Fix: Removed the useEffect cleanup return from hud_glitch handler. Added
   setGlitchClass(null) + timer clear to scene_change handler as safety net.
   Dependency array of scene_change useEffect now includes glitchTimerRef
   and setGlitchClass.

5. BUG 3 FIX — Slow Motion Video
   Files: useGameHudGeneralEffects.ts (2 sites), useGameHudScenarioEffects.ts (1 site)
   Commit: 5757231
   Deployed: Cloudflare Pages version 91abe13c-8da7-4914-be70-36c67a8b5606

   Root cause: No explicit playbackRate set on video elements. High-bitrate
   GCS clips (15-25 Mbps) likely overwhelmed browser decode pipeline.

   Fix: Added defensive video.playbackRate = 1.0 to all 3 video play sites:
     - GCS clip play (useGameHudGeneralEffects.ts)
     - Wildcard scene_video play (useGameHudGeneralEffects.ts)
     - Acecard reveal clip play (useGameHudScenarioEffects.ts)

   Companion fix: FFmpeg re-encode (see item 3 above).
