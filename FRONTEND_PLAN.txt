SECTION 1 — TECH STACK DECISION
--------------------------------------------------------------------------------

USE: Vanilla HTML + CSS + JavaScript (ES2022 modules). No framework, no build step.
     Single HTML file (index.html) + one JS module (app.js) + one CSS file (style.css).

RATIONALE:
  - Zero build tooling = fastest to iterate under deadline.
  - The experience is a single-page, single-session app with no routing.
  - Web Audio API, getUserMedia, WebSocket, and canvas are all native browser APIs.
  - No React state reconciliation overhead touching the media pipeline.
  - Easily deployable as a static site to Cloudflare Pages / Netlify / GitHub Pages.

DO NOT use: React, Vue, Svelte, TypeScript on the frontend, any bundler (Vite/webpack).
            Keep it as close to raw HTML/CSS/JS as possible.

SECTION 5 — WEBSOCKET CLIENT
--------------------------------------------------------------------------------

ENDPOINT: wss://liminal-sin-server-1071754889104.us-west1.run.app

CONNECT ON: user clicks [PLAY] (after permissions granted).

RECONNECT: simple 3-attempt backoff (1s, 3s, 5s). After 3 failures, show
           "Connection lost. Refresh to try again."

MESSAGE ROUTING (incoming):

  switch (data.type) {
    case 'session_ready':     → Phase 2 (start credits sequence)
    case 'agent_speech':      → decode PCM base64, play via Web Audio API
    case 'agent_interrupt':   → stop currently playing NPC audio immediately
    case 'trust_update':      → update trust/fear bars (smooth transition)
    case 'scene_change':      → store mediaId + triggerType + timeoutSeconds; play Morphic media
    case 'scene_image':       → show Imagen-generated still (base64 data) ONLY if no Morphic file
                                 for this mediaId. Do NOT replace a valid Morphic still with Imagen output.
    case 'scene_video':       → same — use Veo GCS URI ONLY if no Morphic clip for mediaId.
    case 'video_gen_started': → show a subtle "generating..." indicator if desired (optional)
    case 'hud_glitch':        → trigger glitch CSS animation
    case 'slotsky_trigger':   → route by anomalyType (see Section 10)
    case 'card_discovered':   → show card overlay for cardId
    case 'dread_timer_start': → (NOOP — dread timer is invisible; heartbeat SFX already started)
    case 'game_over':         → show game over screen (kill all timers, stop audio)
    case 'good_ending':       → show good ending screen (kill all timers, stop audio)
    case 'wildcard3_trigger': → begin wildcard good ending CSS treatment (see Section 10)
    case 'acecard_keyword_timer_start': → start heartbeat SFX escalation loop (30s)
    case 'acecard_reveal_start':        → clear heartbeat SFX; play acecard_reveal_01.mp4 clip
    case 'card_pickup_02_ready':        → show card_pickup_02.png still + card2 overlay (15s); restart urgent SFX
    case 'npc_idle_nudge':    → (data handled by overlay_text; npc_idle_nudge itself can be ignored or used to escalate visual)
    case 'overlay_text':      → show text overlay for durationMs
    case 'hint':              → show hint overlay for 3s
    case 'player_speak_prompt': → Phase 4: show trust meter; activate mic/cam indicators; show talk prompt
    case 'autoplay_advance':  → (informational; frontend may optionally suppress duplicate scene for smooth UX)
    case 'audience_update':   → (informational; no visual output needed)
  }

INCOMING MESSAGES:
  All messages are JSON. Parse with try/catch. Discard unknown types silently.

OUTGOING MESSAGES:
  Frontend sends:
    { type: 'intro_complete' }                     ← after credits sequence ends
    { type: 'player_speech', audio: base64PCM }    ← mic stream chunks (see Section 6)
    { type: 'player_frame',  jpeg: base64JPEG }    ← webcam 1 FPS (see Section 7)
    { type: 'card_collected', cardId: 'card1'|'card2' }
    { type: 'hallway_pov_02_ready' }               ← when hallway_pov_02 still is displayed
    { type: 'acecard_reveal_complete' }            ← when acecard_reveal_01.mp4 clip ends

SECTION 6 — MICROPHONE INPUT (player_speech)
--------------------------------------------------------------------------------

API: navigator.mediaDevices.getUserMedia({ audio: true, video: false })
     (camera is requested separately — see Section 7)

STREAM HANDLING:
  1. Request mic in onboarding on [GRANT PERMISSIONS] click.
  2. Connect mic stream to a ScriptProcessorNode (or AudioWorkletNode) with buffer 2048.
  3. On each audio chunk: resample to 16kHz PCM16LE (required by Gemini Live).
  4. Convert PCM buffer to base64 string.
  5. If connected AND jasonReadyForPlayer (set on player_speak_prompt event):
       ws.send(JSON.stringify({ type: 'player_speech', audio: base64PCM }))

PCM RESAMPLING:
  Gemini Live expects: audio/pcm;rate=16000, 16-bit signed little-endian.
  Browser mic is typically 44100Hz or 48000Hz float32.
  Resample by: downsampling float32 at sampleRate to 16000, then convert to Int16Array,
  then Buffer.from(int16Array.buffer) → base64.
  Simple downsample: keep every N-th sample where N = Math.round(sampleRate / 16000).
  This is good enough for voice. No high-quality resampler needed.

SEND RATE: Every ~100ms (chunks of ~1600 samples at 16kHz). Match ScriptProcessorNode buffer
           size to get ~100ms chunks naturally.

GATE: Do NOT send player_speech until player_speak_prompt event is received.
      (The backend also gates, but frontend gating prevents wasted bandwidth during intro.)

MICROPHONE INDICATOR:
  Show a small dot in the corner (green = active, gray = muted).
  Only show after player_speak_prompt. Hidden during Phases 1–3.

SECTION 7 — WEBCAM FRAME CAPTURE (player_frame)
--------------------------------------------------------------------------------

API: navigator.mediaDevices.getUserMedia({ video: { width: 640, height: 360 }, audio: false })

STREAM HANDLING:
  1. Request camera in onboarding on [GRANT PERMISSIONS] click.
  2. Connect video stream to a hidden <video id="cam-source"> element.
  3. Start frame capture loop on player_speak_prompt event.
  4. Every 1000ms: draw cam-source to a 320x180 offscreen canvas.
  5. canvas.toBlob('image/jpeg', 0.7) → convert blob to base64.
  6. ws.send(JSON.stringify({ type: 'player_frame', jpeg: base64JPEG }))

RESOLUTION: 320x180 JPEG at 70% quality is sufficient for GM vision detection.
            Beyond 640x360 wastes bandwidth with no AI quality gain.

GATE: Do NOT send player_frame until player_speak_prompt event is received.

CAMERA INDICATOR:
  Same approach as mic indicator. Green dot when sending. Only after player_speak_prompt.

WILDCARD VISION FRAME:
  The "latest player frame" is also captured automatically during the 1FPS loop.
  The backend uses it for wildcard_vision_feed image editing (Imagen img2img).
  No special frontend action needed — the regular 1FPS frames are enough.

SECTION 8 — NPC AUDIO PLAYBACK (agent_speech)
--------------------------------------------------------------------------------

INCOMING: { type: 'agent_speech', agent: 'jason'|'audrey', audio: base64 }
         audio = base64-encoded PCM (16kHz, 16-bit, mono)

PLAYBACK:
  1. Maintain an AudioContext (Web Audio API).
  2. On each agent_speech chunk: decode base64 → Int16Array → Float32Array → AudioBuffer.
  3. Append AudioBuffer to a playback queue.
  4. Chain playback: when current source ends, start next queued buffer.
  5. This creates seamless continuous speech from fragmented Gemini Live audio chunks.

INTERRUPT (agent_interrupt):
  On { type: 'agent_interrupt', agent: 'jason' }:
  - Cancel all Jason audio in the queue immediately.
  - Stop the currently playing source.
  - Clear the queue.
  - Do NOT interrupt Audrey if she has a queued message.

VOLUME:
  Jason: 1.0
  Audrey: 0.6 (slightly lower — she is distant and muffled by lore)

VOICE IDENTIFICATION:
  Jason audio plays immediately at full volume on arrival.
  Audrey audio plays at 60% volume — apply a gentle low-pass filter (cutoff ~2500Hz)
  to simulate the muffled, echoing nature of her voice per the character brief.

SILENCE DETECTION:
  Not required on frontend. Backend timer handles idle nudges.

SECTION 9 — SCENE/MEDIA HANDLING
--------------------------------------------------------------------------------

MORPHIC MEDIA STRATEGY (primary):
  When scene_change or scene_video arrives with a mediaId:
  1. Check if mediaId has a clip (use MORPHIC_MEDIA_IDS set or try-catch fetch).
  2. If clip exists: set scene-video.src = `${GCS_BASE}/clips/${mediaId}.mp4`, play.
  3. When video ends (or for chained_auto steps): show `${GCS_BASE}/stills/${mediaId}.png` as still.
  4. If no clip exists for mediaId: show still directly from `${GCS_BASE}/stills/${mediaId}.png`.

GENERATED MEDIA (fallback / wildcard only):
  When scene_image arrives with base64 data:
    - This is ONLY used for wildcard live-generation scenes (wildcard_vision_feed, wildcard_good_ending, etc.)
    - Do NOT replace a valid Morphic still with an Imagen-generated image during scripted scenes.
    - Implementation: if (mediaId in MORPHIC_MEDIA_IDS) { ignore scene_image data; GCS still already showing }
    - For wildcard mediaIds (NOT in MORPHIC_MEDIA_IDS): render base64 data to canvas or <img src="data:image/jpeg;base64,...">

MORPHIC_MEDIA_IDS constant (hardcode in app.js):
  const MORPHIC_MEDIA_IDS = new Set([
    'tunnel_darkness_01', 'tunnel_flashlight_01', 'tunnel_generator_01',
    'card_joker_01', 'card_pickup_01', 'card_pickup_02',

    'park_walkway_02', 'park_liminal_01', 'shaft_maintenance_01',

    'elevator_inside_02', 'hallway_pov_01', 'hallway_pov_02', 'acecard_reveal_01'
  ]);

CLIP PLAYBACK:
  scene-video.src = `${GCS_BASE}/clips/${mediaId}.mp4`
  scene-video.play()
  scene-video.addEventListener('ended', onClipEnd)

  onClipEnd:
    - Fade video to 0 opacity.
    - Fade still (scene-still) to 1 opacity with `${GCS_BASE}/stills/${mediaId}.png` if it exists.
    - If triggerType === 'chained_auto': do nothing more — next autoplay_advance will come from backend
                                         or GM will fire next scene_change.
    - If triggerType === 'hold_for_input': hold on still, fire glitch_low SFX.

STILL DISPLAY:
  scene-still.src = `${GCS_BASE}/stills/${mediaId}.png`
  scene-still.style.opacity = 1

HALLWAY_POV_02 TRIGGER:
  When scene_change arrives with mediaId === 'hallway_pov_02':
    After the still is displayed and loaded:
      ws.send(JSON.stringify({ type: 'hallway_pov_02_ready' }))

VEO VIDEO PLAYBACK (wildcard only):

    This is ONLY for wildcard scenes (wildcard_vision_feed, wildcard_game_over, wildcard_good_ending).

  or the video can be downloaded by the backend and re-served as a data URI.
  This is a known open item — clarify with user before frontend deployment.

SECTION 12 — PHASE STATE MACHINE
--------------------------------------------------------------------------------

Maintain a single state variable: let phase = 'onboarding'
Transitions drive UI show/hide behavior.

phase: 'onboarding'
  - Show: #screen-onboarding
  - Buttons active: #btn-grant, then #btn-play (after permissions)
  - WS: NOT connected yet

phase: 'connecting'
  - Show: black screen (loading state)
  - WS: connecting
  - On session_ready → phase: 'credits'

phase: 'credits'
  - Show: #screen-credits
  - Auto-advance through credits lines (2s fade + 2s hold + 1s fadeout each)
  - SFX: wind_ambient begins; floor_crack fires on last credit exit
  - On sequence end: send intro_complete; phase: 'darkness'

phase: 'darkness'
  - Show: all black (scene-still hidden, scene-video hidden)
  - Audio only: Jason's landing monologue
  - SFX: drip_loop, structural_hum begin; wind_ambient fades slightly
  - On player_speak_prompt: phase: 'interaction'

phase: 'interaction'
  - Show: scene-still or scene-video (as media arrives)
  - Trust meter visible (fade in)
  - Mic + cam active
  - Card overlays enabled
  - All WS events routed normally
  - Sub-phases (flashlight, generator, park, maintenance) handled by scene_change events
  - On game_over: phase: 'game_over'
  - On good_ending: phase: 'good_ending'

phase: 'game_over'
  - Kill all timers, stop all SFX
  - Fade to: show #screen-game-over
  - WS: do not close (allow backend to log session)

phase: 'good_ending'
  - Kill all timers, stop non-ending SFX
  - Fade to: show #screen-good-ending
  - WS: do not close

ON PLAY AGAIN (#btn-play-again):
  - Close WS
  - location.reload() — simplest restart
